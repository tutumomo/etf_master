# ETF_TW 持倉重配置建議工作流程

## 何時使用
- 使用者要求基於當前持倉給出投資建議
- 使用者問「現在有現金應該怎麼配置」
- 使用者問「持倉比例健不健康」
- 定期檢視（週報/月報）需附帶重配置建議
- 使用者問「如果回檔優先買什麼」「逢低加碼哪一檔」→ 使用回檔加碼優先序流程（Step 7）
- 使用者問「某檔已賺 X 元可不可以賣」「下週一開市要不要獲利了結」→ 使用獲利了結檢核流程（Step 8）

## 核心原則

1. **真相分級**：持倉必須從 live state 讀取，不用記憶或推測
2. **知識庫驅動**：建議依據 wiki/investment-strategies.md 的原則，非主觀判斷
3. **可執行**：每個建議必須附帶具體標的、股數、金額
4. **風險透明**：每個建議同時標注風險/限制

## 步驟

### Step 1：讀取持倉真相（5 個必要 state 檔）

```
instances/etf_master/state/positions.json        — 持倉明細
instances/etf_master/state/portfolio_snapshot.json — 總權益/現金
instances/etf_master/state/strategy_link.json    — 策略狀態
instances/etf_master/state/market_intelligence.json — RSI/MACD/BB/動能
instances/etf_master/state/market_context_taiwan.json — 體制判讀
```

**補充 state 檔（非必要但大幅提升建議品質）：**
```
instances/etf_master/state/major_event_flag.json     — 重大事件旗標
instances/etf_master/state/decision_quality.json     — 決策品質分數
instances/etf_master/state/auto_preview_candidate.json — 最新候選標的
instances/etf_master/state/worldmonitor_snapshot.json  — 全球風險壓力指數
```

用 `execute_code` + `json.load()` 讀取，不可以用 `read_file()`（行號格式會炸 JSON 解析）。

**etfs.json 結構注意**：`data/etfs.json` 是 dict（keyed: `meta`, `etfs` (dict by symbol), `categories`），不是 list。部分較新 ETF（如 00919, 00922, 00923, 00939）可能不在 etfs.json 中，需從 market_intelligence.json 的 `intelligence` dict 取得 RSI/群組資料。

### Step 2：分類計算當前配置

將每個持倉歸入四大類：

| 類別 | 定義 | 目標範圍（核心累積策略） |
|------|------|----------------------|
| 市值型核心 | 0050, 006208, VOO 等 | 40-50% |
| 高股息 | 00878, 00919, 0056 等 | 20-30% |
| 債券型 | 00679B, 00720B, 00687B 等 | 10-15% |
| 現金/備用 | 現金餘額 | 15-20% |

計算公式：
- 類別市值 = 該類別所有持倉市值之和
- 類別佔比 = 類別市值 / 總資產（total_equity）

### Step 3：診斷問題（對比知識庫原則）

常見問題模式：

| 問題 | 偵測條件 | 對應知識庫原則 |
|------|---------|-------------|
| 現金過高 | 現金佔比 >30% | Vanguard「Time in the Market」+ 機會成本 0.38%/年 |
| 核心不足 | 市值型 <30% | 核心衛星「核心≥50%」+ Boglehead「只投資標竿指數」 |
| 債券太輕 | 債券型 <5% | 清流君「全球分散 + 暖男策略」+ 0407 教訓 |
| 單檔過度集中 | 單 ETF >總資產 25% | 風控硬限制 + 陷阱 8「成分股重疊」偽分散 |
| 追高風險 | RSI >70 且 BB 位置>上軌 | 陷阱 1「逢低買進」反模式 + 紀律原則 |
| 停滯現金 | 現金 >緊急備用金需求 | 策略 9「緊急備用金優先」 |

### Step 4：設定目標配置

根據策略 + 體制決定目標：

| 體制 | 市值型 | 高股息 | 債券型 | 現金 |
|------|--------|--------|--------|------|
| balanced_bullish | 45% | 25% | 10% | 20% |
| strongly_bullish | 50% | 20% | 5% | 25% |
| cautious | 30% | 25% | 20% | 25% |
| bearish | 20% | 20% | 25% | 35% |

**股齡修正**：110 - 年齡 = 股票總比例（市值+高股息），債券+現金 = 餘額

### Step 5：產出具體操作建議

每個操作必須包含：
- **標的代碼 + 名稱**
- **買/賣/持有**
- **具體股數**（零股也要算）
- **預估金額**
- **理由**（引用知識庫哪一條原則）
- **風險提醒**（RSI/MACD/BB 狀態）

優先序：
1. 加碼市值型核心（最高優先）
2. 加碼債券防禦
3. 調節高股息（多則停、少則補）
4. 降低現金比例

### Step 6：總結一句話

將整個建議濃縮為一行，格式：
「把現金從 X% 降到 Y%，多出來的 N 元優先補 [類別]，其次補 [類別]，讓股債比例從 A:B 回到 C:D。」

### Step 7：回檔加碼優先序（使用者問「回檔先買誰」時使用）

此步驟與一般重配置不同：不是從「類別配置」出發，而是從「低配 + 已驗證」角度排序。

**排序邏輯（3 因子加權）：**

| 因子 | 權重 | 說明 |
|------|------|------|
| 低配嚴重度 | 40% | 佔比與目標的差距越大，加碼優先度越高 |
| 已驗證績效 | 30% | 目前持倉中報酬率最好的標的，回檔時加碼勝率更高（已驗證 ≠ 保證未來） |
| RSI 安全邊際 | 30% | RSI>70 者需等回檔，RSI<50 者可立即進場 |

**執行規則：**
1. **不加碼已達上限的標的**（佔比≥25% 或逼近 30%）
2. **已驗證 + 低配 = 最佳回檔加碼標的**（如 00878: 報酬+11% 但佔比僅 1.9%）
3. **RSI 超買者標「等回檔」而非「現在買」**，給出回檔進場價位參考（預估 RSI 回到 65 以下的價位）
4. **RSI 接近超賣者可「不等回檔」直接分批進場**（如 00679B RSI 42.8）
5. **每次加碼後現金仍需保留 ≥20%**（cautious 體制下保留 25%）
6. **明確列出「不優先加碼」的名單及理由**

**輸出格式：**
```
### 🥇 第一優先：[代碼 名稱]
| 指標 | 數值 | 判斷情緒圖標 |
理由 + 建議加碼金額/目標佔比

### ❌ 不優先加碼
| 標的 | 現佔比 | 不加碼理由 |

### 💰 資金分配建議
| 標的 | 加碼金額 | 目標佔比 | 預估股數 |

### ⏱ 執行時機
- 現在可進場 vs 等回檔到什麼價位
```

### Step 8：獲利了結檢核（使用者問「賺了可不可以賣」時使用）

此步驟處理單一持倉的賣出/部分賣出問題，不能只看未實現損益金額。

**必做檢查：**
1. **先確認標的角色**：若是 0050/006208 這類市值型核心，不因小幅獲利就預設全賣；核心部位應優先考慮「部分調節」或「停止加碼」。
2. **計算真實獲利率與交易成本**：用工具計算 `市值 = 股數 × 現價`、`推估成本 = 市值 - 未實現損益`、`報酬率 = 損益 / 成本`；賣出 ETF 要估證交稅與手續費，回報「扣成本後約剩多少」。
3. **檢查配置缺口**：若該類別本來低配（例如市值型核心 < 目標），不建議全賣；若該類別超配或單檔集中，才提高賣出優先度。
4. **檢查技術熱度**：RSI > 70 可支持「部分獲利了結」，但不是全清理由；RSI 未過熱則偏向續抱。
5. **給具體股數方案**：至少提供「不動 / 賣一半 / 賣固定股數」三種判斷，但若使用者要求正式下單，不再給選項，直接走交易 SOP。
6. **交易規則提醒**：下週一開市賣出應在交易時段送限價 ROD；零股可賣，未成交不跨日保留。

**決策口徑：**
- 核心 ETF + 組合低配 + 小幅獲利：不建議全賣，可賣 1/3～1/2 或不動。
- 核心 ETF + RSI 過熱：可部分賣出，保留底倉。
- 衛星/題材 ETF + 單檔超配 + RSI 過熱：可較積極獲利了結。
- 只因「看到 +X 元」想賣：先把 X 換算成報酬率與稅費後再判斷。

**輸出格式：**
```
可以/不建議全賣，原因：
- 報酬率與扣費後獲利
- 該標的在組合中的角色與配置是否低配/超配
- RSI/技術熱度
建議：賣 N 股 / 賣一半 / 暫不動
```

## 注意事項

- **不保證獲利**：結尾必須附風險聲明
- **交易時段**：提醒操作需在 09:00-13:30 執行
- **零股規則**：台灣零股限價 ROD，不可改價（只能減量或取消重掛）
- **RSI 門檻**：RSI>70 不追高；RSI<40 可考慮加碼
- **殖利率陷阱**：高殖利率 ≠ 防禦力（0407 教訓）
- **繁體中文**：全篇輸出必須繁體
