# 交易流程說明

## 核心流程

所有交易相關動作必須遵循以下流程：

```
validate → preview → confirm → execute
```

## 模式切換與 Dashboard 對齊

當使用者要求切換交易模式（paper / live）時，ETF_TW 應依下列方式處理：

1. 將口語請求映射到正式模式指令（`mode paper` / `mode live`）
2. 先確認 `trading_mode.json` 已更新
3. 再確認 dashboard 顯示的模式是否與 mode state 一致
4. 若兩者一致，視為本次模式切換已完成必要對齊
5. 若使用者未另外要求，不必為此再全面核對現金、總資產、所有卡片數值
6. 若 mode state 與 dashboard 模式不一致，才升級為排障問題

### 預設回報格式
- 模式切換結果
- dashboard 是否對齊
- 如有不一致，再說明下一步排查方向

## 步驟詳解

### 1. Validate（驗證）

**目的**：檢查下單條件是否合理

**檢查項目**：
- 標的是否有效（ETF 代號是否存在）
- 數量是否合理（非負數、非過大）
- 價格是否合理（非 0、非異常值）
- 動作類型是否有效（BUY / SELL）
- 單位是否明確（股 vs 張）

**範例**：
```
使用者：買進 0050 100 股

Validate 結果：
✅ 標的有效：0050.TW
✅ 數量合理：100 股
✅ 動作有效：BUY
✅ 單位明確：股

驗證通過，進入 preview
```

**失敗處理**：
- 標的無效：「0050 不存在，請確認代號」
- 數量不合理：「數量必須為正數」
- 單位不明：「請確認是 股 還是 張」

---

### 2. Preview（預演）

**目的**：估算下單結果與成本

**計算項目**：
- 成交金額（價格 × 數量）
- 手續費（成交金額 × 0.1425%）
- 證交稅（賣出時：成交金額 × 0.3%）
- 總成本 / 總回收
- 持倉變化
- 現金變化

**範例**：
```
使用者：買進 0050 100 股 @ 75 元

Preview 結果：
┌─────────────────────────────────┐
│ 下單摘要                         │
│ 標的：0050 (台灣 50)              │
│ 動作：買進                       │
│ 數量：100 股                      │
│ 價格：75 元                       │
├─────────────────────────────────┤
│ 成本估算                         │
│ 成交金額：7,500 元               │
│ 手續費：10.69 元                 │
│ 證交稅：0 元（買進不收）         │
│ 總計：7,510.69 元                │
├─────────────────────────────────┤
│ 持倉變化                         │
│ 目前持倉：0 股                    │
│ 交易後持倉：100 股                │
├─────────────────────────────────┤
│ 警告事項                         │
│ ⚠️ 目前為 paper 模式             │
│ ⚠️ 價格為預估，實際成交可能不同 │
└─────────────────────────────────┘

是否執行？(confirm / cancel)
```

**無法安全估算時**：
- 資料不新鮮：「目前價格可能已過時，建議觀望」
- 流動性不足：「該標的流動性較低，成交可能不順」
- 成本不明：「無法估算完整成本，建議確認券商費用」

---

### 3. Confirm（確認）

**目的**：確認使用者意圖與模式

**確認項目**：
- 下單意圖是否明確
- 模式是否正確（paper / sandbox / live）
- 大額交易是否需要額外確認
- 模式切換是否需要確認

**確認語句範例**：
- 「確認執行 paper trade 嗎？」
- 「此為大額交易（超過總持倉 50%），請輸入 CONFIRM LARGE ORDER 確認」
- 「從 paper 切換至 sandbox 需要明確確認，是否繼續？」

---

### 4. Execute（執行）

**目的**：實際執行交易

**執行模式**：

#### Paper Trade（模擬）
- 更新模擬持倉
- 更新交易紀錄
- 更新現金餘額
- 記錄交易日誌

**範例**：
```
✅ 買進成功！
0050 x 100 股 @ 75.00 元

持倉更新：
- 0050: 100 股（平均成本 75.00 元）

現金餘額：992,489.31 元（原 1,000,000 元）

交易紀錄已保存至 paper_ledger.md
```

#### Sandbox（模擬券商）
- 需明確確認
- 使用券商沙盒環境
- 記錄真實 API 呼叫
- 不影響真實帳戶

#### Live（真實交易）
- **目前不支援**
- 未來需額外 safeguards
- 需明確授權
- 需額外確認步驟

## 特殊情境處理

### 1. 單位混淆

**情境**：使用者說「400 張 0050」但帳戶只有 25k 元

**處理**：
```
⚠️ 單位混淆警告！

您輸入：400 張 0050 = 40,000 股
目前帳戶總值：約 25,000 元
單筆交易價值：約 3,000,000 元

差異：120 倍（超過 10 倍閾值）

請確認：
A. 您確實要買 400 張（300 萬）
B. 您實際想買 400 股（3 萬）
C. 取消交易

請回覆 A / B / C
```

### 2. 大額交易

**情境**：單筆交易超過總持倉 50%

**處理**：
```
⚠️ 大額交易警告！

單筆交易價值：1,500,000 元
目前總持倉：2,000,000 元
占比：75%（超過 50% 閾值）

此交易將大幅改變持倉配置，請確認：
- 是否了解集中度風險
- 是否有意為之

請輸入 "CONFIRM LARGE ORDER" 以繼續
```

### 3. 訊號衝突

**情境**：技術指標顯示買進，但基本面顯示風險

**處理**：
```
⚠️ 訊號衝突！

技術面：
- RSI: 28（超賣，買進訊號）
- MA20: 股價跌破（賣出訊號）

基本面：
- 本週財報可能不如預期
- 產業趨勢轉弱

建議：觀望，或減碼執行

是否繼續？(proceed / wait / reduce)
```

## 錯誤處理

### 常見錯誤

| 錯誤 | 處理方式 |
|------|----------|
| 標的無效 | 停止，提示正確代號格式 |
| 數量為負 | 停止，要求正數 |
| 現金不足 | 停止，顯示目前現金 |
| 持倉不足（賣出） | 停止，顯示目前持倉 |
| 價格為 0 | 停止，要求有效價格 |
| 單位不明 | 暫停，要求確認股/張 |

### 錯誤回應範例
```
❌ 驗證失敗

原因：現金不足
需要：7,510.69 元
目前：5,000 元

請增加現金或減少交易數量
```
