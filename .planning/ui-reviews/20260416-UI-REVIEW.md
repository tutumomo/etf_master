---

## Top 3 Priority Fixes

1. **Reduce UI "Flicker" from Reloads** — User impact: High (Annoyance) — Transition simple state changes (Watchlist, Mode Switch) from `window.location.reload()` to DOM-only updates. Remove the arbitrary 1000ms delay in `initIntelligence`.
2. **Mobile Table Responsiveness** — User impact: Medium (Usability) — The 11-column Positions table overflows on mobile. Add `overflow-x: auto` to table containers or implement a card-based view for `< 640px`.
3. **Theming Debt Consolidation** — User impact: Low (Consistency) — Move the 80+ hardcoded `rgba` and hex colors in `overview.html` into CSS variables in `base.html` to ensure unified brand colors across the whole dashboard.

## Detailed Findings

### Pillar 1: Copywriting (4/4)
- **High-Quality Context:** Descriptions like "本地資產儀表板已開始承接 agent 的高頻展示資訊" provide great situational awareness.
- **Action Labels:** Specific labels like "切到 LIVE" and "立即同步資料" are better than generic "Switch" or "Sync".
- **Risk UX:** Trade ticket checkbox ("我已確認上述交易細節與風險提示...") uses professional legal-adjacent phrasing.

### Pillar 2: Visuals (3/4)
- **Decision Matrix Clarity:** The Rule/AI/Consensus blocks are clearly differentiated by color (Blue/Purple/Green).
- **Chart Integration:** TradingView lightweight charts are integrated well into table rows without breaking the layout.
- **Visual Feedback:** "載入中..." and "資料尚未建立" states prevent the UI from looking "broken" while data is fetching.
- **Negative:** Page-level refresh on minor actions breaks the "Control Center" immersion.

### Pillar 3: Color (3/4)
- **Palette Consistency:** Good use of `good` (#35d49a) for profit/buy and `bad` (#ff6b81) for loss/sell.
- **Hardcoding Issue:** Found 87 instances of hardcoded colors in `overview.html`.
  - Reference: `rsiColor = rsi > 70 ? '#ff6b81' : ...` (line 980)
  - Reference: `style="background:rgba(53,212,154,0.12);"` (line 747)
- Suggestion: Use CSS variables (e.g., `rgba(var(--good-rgb), 0.12)`) instead.

### Pillar 4: Typography (4/4)
- **Data Readability:** `.mono` class correctly applies `tabular-nums` (line 228 in `base.html`), making price columns align perfectly.
- **Weight Contrast:** Bold headings and muted labels create a clear reading path.

### Pillar 5: Spacing (3/4)
- **Inconsistent Implementation:** Layout uses a grid (12 columns), but many individual spacing adjustments are inline.
  - `style="margin: 0 16px 18px 16px;"` (line 33)
  - `style="gap:10px; ..."` (line 72)
- **Improvement:** Define a standard spacing scale (4px, 8px, 12px, 16px, 24px) in `base.html` and use utility classes.

### Pillar 6: Experience Design (3/4)
- **Trade Ticket Safety (Excellent):** The 3-stage flow (Preview -> Confirm -> Submit) is a highlight. Locking the form during Preview (line 1133) prevents "editing under the hood" after checking risk.
- **State Handling:** Good coverage of loading/empty/error states.
- **Latency UX:** The `setTimeout(..., 1000)` on load (line 1245) causes the Consensus board to show "Loading" for 1 second even if the API is fast. This should be reduced or removed.
