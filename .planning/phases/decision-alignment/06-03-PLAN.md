# 06-03-PLAN: 實作 Dashboard 反應式連動 (app.py)

## Objective
實作 Dashboard 的反應式連動，當策略更新後自動觸發仲裁重掃描。

## Context
目前 Dashboard 修改策略後，使用者需要手動刷新或等待一段時間才能看到仲裁結果的更新。反應式連動能提高即時性。

## Tasks
- [ ] 1. 實作「清除舊建議」邏輯：在重新生成前清除舊的 `decision_consensus.json` @auto
- [ ] 2. 修改 `/api/strategy/update`：成功更新後立即觸發 `_run_consensus_rescan()` @auto
- [ ] 3. 確保 `_run_consensus_rescan()` 能夠非同步執行以避免阻塞請求 @auto
- [ ] 4. 驗證策略變更後仲裁結果是否自動重新整理 @auto

## Success Criteria
- 成功更新策略後，仲裁舊結果會被立即刪除。
- `/api/strategy/update` 回應後，後台開始重新掃描。
- Dashboard 能夠呈現最新的「對齊後」建議。
