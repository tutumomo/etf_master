# 06-01-PLAN: Rule Engine 動態權重矩陣實作

## Objective
在規則引擎 (`run_auto_decision_scan.py`) 中實作動態權重矩陣，取代硬編碼的評分比例，使其能根據當前的 `base_strategy` (例如：收益優先、核心累積) 自動調整評分邏輯。

## Context
目前的 `run_auto_decision_scan.py` 使用固定的基礎分與加分邏輯。為了實現「決策對齊」，不同策略應該對不同的 ETF 群組（Core, Income, Defensive, Growth）有不同的偏好權重。

## Tasks
- [x] 1. 定義權重矩陣查詢表 (Lookup Table) @auto
- [x] 2. 修改 `decide_action` 函數讀取動態權重 @auto
- [x] 3. 更新評分邏輯以應用權重 @auto
- [x] 4. 驗證不同策略下的評分結果 @auto

## Success Criteria
- 移除 `run_auto_decision_scan.py` 中硬編碼的 `group_base` 與其他權重。
- 策略切換後，執行 `run_auto_decision_scan.py` 產出的分數能反應權重差異。
- 支援 `收益優先`、`核心累積`、`平衡配置`、`防守保守` 等主要策略。
