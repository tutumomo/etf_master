# Phase 3: 持倉交易票據 UI Summary

本階段完成了持倉交易介面的現代化與流程安全加固。透過摺疊式卡片、新手導引、內嵌交易票據以及嚴謹的三段式交易 API，提升了 ETF_TW Dashboard 的專業度與操作安全性。

## 核心交付產物

### 1. Dashboard UI 整體優化 (TICKET-06, TICKET-07)
- **卡片摺疊機制**：實現了所有 Dashboard 卡片的點擊摺疊功能，使用者可以更自由地配置視野，重點關注當前需要的資訊。
- **新手導引與 Tooltips**：在關鍵指標（總資產、交易模式、盤感輔助等）旁加入問號小提示，降低新手的學習門檻。

### 2. 持倉交易入口與票據 UI (TICKET-01, TICKET-02)
- **交易按鈕入口**：在持倉快照表格中新增操作欄位與「交易」按鈕，主列保持整潔。
- **內嵌式票據 (Inline Ticket)**：點擊交易按鈕後在列下方直接展開交易區塊，無需切換頁面即可進行快速操作。

### 3. 三段式交易安全流程 (TICKET-03, TICKET-04, TICKET-05)
- **Preview -> Confirm -> Submit**：實作了嚴格的交易階段流轉。使用者必須先通過預覽、勾選確認後才能送出委託。
- **後端風控整合**：預覽與送單 API 均深度串接 `pre_flight_gate`，進行數量（張/股）、庫存、交易時段與集中度（sizing_engine）的即時檢查。

## 技術進展與決策

- **UI 模式**：採用展開層 (Drawer/Inline Detail) 模式處理交易票據，這在小螢幕或多工處理時比彈窗 (Modal) 更具優勢。
- **狀態管理**：前端 JS 強制在參數變動時重置交易狀態，防止預覽後的參數竄改。
- **安全性**：後端不信任前端預覽結果，在 `/api/trade/submit` 階段二次執行所有 `pre_flight_gate`。
