Structure:
1. `compute_trade_pnl(entry_price: float, exit_price: float, action: str) -> float`
   - action "buy" (or "preview_buy"): (exit - entry) / entry
   - action "sell" (or "preview_sell"): (entry - exit) / entry (profits from price decline)

2. `compute_metrics(pnl_list: list[float]) -> dict`
   - win_rate: count(pnl > 0) / len(pnl_list), returns None if empty
   - max_drawdown: compute equity curve (cumsum of pnl), find maximum peak-to-trough decline
   - sharpe_ratio: mean(pnl) / std(pnl) * sqrt(252) if len >= 2, else None
   - avg_pnl: mean of pnl_list

3. `evaluate_quality_gate(win_rate: float | None, max_drawdown: float | None) -> bool`
   - Returns True only if win_rate is not None AND win_rate >= 0.5 AND max_drawdown is not None AND max_drawdown <= 0.15

4. `fetch_price(symbol: str, date_str: str, price_cache: dict) -> float | None`
   - Uses `price_cache` dict to avoid redundant yfinance calls
   - Calls `yf.download(ticker, start=date_str, end=end_date_str, auto_adjust=True, progress=False)`
   - ticker = symbol + ".TW" unless symbol starts with "006" → use ".TWO"
   - Returns Close price on date_str, or None on failure
   - On yfinance error: print warning, return None (never raise)

5. `run_backtest(records: list[dict], holding_days: int = 5, price_fetcher=None) -> dict`
   - `price_fetcher` is injected for testing (signature: `(symbol, date_str) -> float | None`)
   - Filters records where action in ("preview_buy", "preview_sell", "buy", "sell")
   - For each: fetch entry price on recorded_at date, fetch exit price on recorded_at + holding_days calendar days
   - Builds trade list: `[{"symbol", "action", "entry_date", "exit_date", "entry_price", "exit_price", "pnl"}]`
   - Skips trades where price fetch returns None
   - Calls compute_metrics on pnl list

6. Output JSON schema:
```json
{
  "total_decisions_evaluated": 0,
  "tradeable_decisions": 0,
  "trades_with_price_data": 0,
  "trade_details": [...],
  "win_rate": null,
  "max_drawdown": null,
  "sharpe_ratio": null,
  "avg_pnl": null,
  "avg_holding_days": 5,
  "quality_gate_passed": false,
  "quality_gate_reason": "insufficient data",
  "last_updated": "ISO8601"
}
```

7. `main()` CLI: `python backtest_decision_outcomes.py [--holding-days N]`
   - Reads `get_state_dir() / "ai_decision_outcome.jsonl"`
   - Writes `get_state_dir() / "backtest_results.json"`
   - Prints: `[backtest] {N} trades evaluated. win_rate={x:.2f}, max_drawdown={x:.2f}, gate_passed={True/False}`

Dependencies: `yfinance`, `numpy` (for sharpe), `datetime`, `zoneinfo` — all available in ETF_TW venv.
