Create `skills/ETF_TW/scripts/run_paper_stress_test.py`.

Architecture:
1. Define `check_cycle_orders(orders: list[dict]) -> dict` — pure function, checks a single cycle's order list for safety violations
   - Returns: `{"ghost_orders": list, "unit_confusion": list, "order_ids": list}`
   - Ghost order: `order.get("broker_order_id") in (None, "", "null") and not order.get("verified", True)`
   - Unit confusion: `order.get("lot_type") == "board" and order.get("quantity", 0) % 1000 != 0`

2. Define `run_stress_test(cycles: int, scan_fn=None) -> dict` — orchestrator
   - `scan_fn` is injected (for testing with mocks); default loads from `auto_preview_candidate.json` + `decision_log.jsonl`
   - Runs `cycles` iterations, calls `check_cycle_orders` on each
   - Tracks: `ghost_orders_detected` (count), `unit_confusion_detected` (count), all `order_ids` seen
   - After all cycles: check for duplicate order_ids (same order_id in more than one cycle)
   - Computes `tier_distribution` across all orders
   - Sets `stress_test_passed = True` only when ghost_orders_detected == 0 AND unit_confusion_detected == 0 AND no duplicates

3. Output JSON schema:
```json
{
  "cycles_run": 10,
  "total_orders_seen": 0,
  "ghost_orders_detected": 0,
  "unit_confusion_detected": 0,
  "duplicate_order_ids": [],
  "tier_distribution": {"tier1": 0, "tier2": 0, "tier3": 0},
  "stress_test_passed": true,
  "failure_reasons": [],
  "last_updated": "ISO8601"
}
```

4. `main()` — CLI entry point: `python run_paper_stress_test.py [--cycles N]` (default N=10)
   - Reads current state from `auto_preview_candidate.json` for each simulated cycle
   - Writes output to `get_state_dir() / "stress_test_report.json"`
   - Prints: `[stress-test] Completed {N} cycles. Passed: {True/False}`

IMPORTANT: The script does NOT actually call Shioaji or submit real orders. It reads existing paper-mode state files to simulate cycles. This is a validation tool, not a trading tool.
