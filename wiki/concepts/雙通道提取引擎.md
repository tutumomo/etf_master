# 雙通道提取引擎

[[graphify]] 的核心技術：結合 AST 靜態分析和 LLM 語義理解兩條提取路徑，實現準確性和覆蓋率的平衡。

## 兩個通道

### Channel A：AST 提取（tree-sitter）

- **零 LLM 開銷**：純語法分析，不消耗 token
- **支援 15 種語言**：Python, JavaScript, TypeScript, Go, Rust, Java, C/C++, etc.
- **提取內容**：函數定義、類別結構、import 關係、呼叫圖
- **邊的置信度**：EXTRACTED（1.0），因為直接從語法樹提取

### Channel B：LLM 語義提取

- **需要 token**：每份文件約消耗數百到數千 token
- **處理範圍**：文檔、論文、圖片、自然語言內容
- **提取內容**：語義關聯、概念連結、設計決策、跨文件關係
- **邊的置信度**：INFERRED（0.6-0.9）或 AMBIGUOUS（低於 0.6）

## 雙通道如何協作

```
原始碼 (.py, .js, .go, ...)
    │
    ├── Channel A (AST) ──→ EXTRACTED 邊 (confidence 1.0)
    │                        函數呼叫、類別繼承、import 關係
    │
文檔 (.md, .pdf, .txt, ...)
    │
    └── Channel B (LLM) ──→ INFERRED/AMBIGUOUS 邊
                             語義關聯、概念連結、設計決策
```

## 深度模式

`--mode deep` 切換到更徹底的提取：
- Channel B 使用更多 LLM 調用
- 產生更多 INFERRED 邊（探索性更強）
- 適合初次建立圖譜或大規模重構後

## 與單一通道的對比

| 方法 | 準確性 | 覆蓋率 | Token 成本 | 發現意外連結 |
|------|--------|--------|-----------|------------|
| 純 AST | 高（程式碼） | 低（僅程式碼） | 零 | 否 |
| 純 LLM | 中 | 高 | 高 | 是 |
| 雙通道 | 高 | 高 | 中 | 是 |
