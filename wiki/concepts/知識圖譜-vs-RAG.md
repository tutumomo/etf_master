# 知識圖譜 vs RAG：兩種知識管理範式

知識圖譜（如 [[graphify]]）和 RAG（Retrieval-Augmented Generation）是兩種讓 LLM 存取外部知識的主要方法。它們有不同的哲學和適用場景。

## 核心差異

| 維度 | 知識圖譜 | RAG |
|------|---------|-----|
| **知識結構** | 節點+邊的圖結構，保留關係 | 向量嵌入，語義相似度 |
| **查詢方式** | BFS/DFS 圖遍歷，最短路徑 | 向量相似度搜索 top-k |
| **持久性** | 建構一次，跨 session 持久 | 每次查詢重新檢索 |
| **Token 效率** | 只讀相關子圖（1/71.5 原始量） | 每次注入相關段落 |
| **可審核性** | 每條邊有置信度標籤 | 向量相似度分數，較不直覺 |
| **意外發現** | 社群偵測發現非顯見連結 | 依賴查詢品質，不會主動發現 |
| **維護成本** | 需要 lint 和一致性檢查 | 索引自動更新，維護較簡單 |

## 選擇指南

### 用知識圖譜當…

- 需要跨文件發現隱藏關聯（社群偵測）
- 需要持久化、可複利的知識累積
- 需要可審核的置信度（EXTRACTED vs INFERRED vs AMBIGUOUS）
- 代碼庫理解、論文語料分析
- 知識需要長期維護和更新

### 用 RAG 當…

- 需要快速搭建，不需持久化
- 知識庫變動頻繁，不適合維護圖結構
- 只需要語義搜索，不需要關係推理
- 短期問答場景

## 混合策略：從圖譜到 Wiki

最佳的知識管理流程可能是兩者的結合（參考 [[知識管理.方法論]]）：

1. **Graphify 探勘**：用圖譜發現隱藏連結和社群結構
2. **選擇性沉澱**：只把 EXTRACTED 和高信心 INFERRED 邊沉澱到 wiki
3. **LLM Wiki 定居**：在 wiki 中長期維護已驗證的知識

## ETF_TW 實測數據（2026-04-14）

> 以下數據來自 Graphify 對 ETF_TW 代碼庫（341 檔案，~96,691 字）的實際提取結果。

### 實測結果摘要

| 指標 | 數值 |
|------|------|
| 總節點數 | 2,654 |
| 總邊數 | 3,164 |
| 發現社群數 | 528 |
| EXTRACTED 邊 | 75% |
| INFERRED 邊 | 25% |
| AMBIGUOUS 邊 | 0% |
| Token 消耗 | 0（規則基底提取） |

### 實測：圖譜查詢效率

```
建圖一次：341 檔案 → 2,654 節點
查詢單位：只讀相關子圖（avg ~37 節點）
對比 RAG：每次注入可能高達數千 token
```

### 實測：隱藏連結發現

Graphify 社群偵測發現了人類未預期的連結：

- `AccountManager` → 依賴 → `BaseAdapter`（帳戶管理與券商適配器的隱藏耦合）
- `Order` 節點連接 Community 0/1/3（委託生命週期跨越三個系統區塊）
- `BaseAdapter` 連接 Community 0/1/10（券商適配器是系統整合中樞）

### 實測：超邊（Hyperedges）

8 個 ETF 生態主題群，自動從文件中推斷：
- `etf_ecosystem` 主題涵蓋：00679B、00878、00929、00922、00892、00713、00637L、00881
- 這是 RAG 純向量搜尋幾乎不可能發現的跨文件主題關聯

### 實測結論

| 維度 | 知識圖譜（實測） | RAG（理論） |
|------|-----------------|-------------|
| 發現隱藏關聯 | ✅ 528 社群自動分組 | ❌ 依賴查詢品質 |
| Token 效率 | ✅ 子圖讀取 | ⚠️ 每次重新檢索 |
| 意外發現 | ✅ Surprising Connections | ❌ 不主動發現 |
| 設定複雜度 | ⚠️ 需 graphify 安裝 | ✅ 向量資料庫即用 |
| 維護成本 | ⚠️ 代碼變更需要重建 | ✅ 增量更新 |

### 混合策略的實際效益

根據 ETF_TW 實測，混合策略（Graphify 探勘 + wiki 沉澱）的效益：

1. **Graphify 探勘**：341 檔案中自動發現 528 個社群，8 個 ETF 生態主題群
2. **選擇性沉澱**：只有 EXTRACTED（75%）和 高信心 INFERRED 邊寫入 wiki
3. **知識累積**：`decision-chain.md` 這類概念頁從 graphify 数据中獲得系統架構依据
