## 重要教訓

### 教訓 33：Shioaji 張/股單位地雷（2026-04-14）
- **根因**：`sinopac_adapter.py` L445 舊版 `lots = qty // 1000 if qty >= 1000 else 1`
  → 100 股輸入被當成 1 張（1000 股）直接送出
- **事故金額**：~217,454 TWD（4/16 T+2 扣款）
- **修復**：submit_order 改為零股友善邏輯（`lots = qty // 1000; if qty % 1000 != 0: lots = 1`）
- **教訓**：TOMO 預設以「股」為單位，任何整股轉換必須明確驗證

### 教訓 34：已實現單查不到（2026-04-14）
- `api.list_trades()` 對已 filled 的訂單回傳空陣列
- Fill 監控必須改用 `api.list_orders()` 或持倉變化對比
- 修復：使用 position snapshot diff 或 `list_orders()` 而非 `list_trades()`
